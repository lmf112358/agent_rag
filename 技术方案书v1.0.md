## 第一章 项目概述

### 1.1 建设背景

广东迪奥技术有限公司作为**国家级专精特新"小巨人"企业**，聚焦PCB行业智慧低碳厂务领域，核心业务涵盖"平台+系统+工程+设备+服务"五位一体的智慧零碳解决方案。随着PCB行业竞争加剧及ESG合规要求趋严，公司在以下环节面临显著效率瓶颈：

- **知识沉淀困境**：20年PCB厂务经验分散在个人电脑与纸质档案中，新人培养周期长，技术复用率低
- **投标响应瓶颈**：单个智慧低碳项目招标书平均200页，技术合规审核耗时3-5天，错漏风险高
- **报价精准度风险**：中央空调系统涉及200+分项报价，人工复核易漏算错算，曾出现因漏计冷却塔基础导致项目亏损案例
- **数据安全红线**：厂务数据涉及客户产线机密，传统SaaS AI工具存在数据出境风险，无法通过客户信息安全审计

**建设目标**：在**1个月内**完成空调领域（高效中央空调节能系统）最小可行Demo（MVP），构建**本地化部署**的AI工具体系，实现知识复用率提升50%、投标审核效率提升50%、报价复核准确率100%，并确保核心数据零泄露。

### 1.2 核心定位

本方案严格遵循**"本地部署、安全可控、分级实施、先易后难"**原则，首期聚焦空调领域，验证"AI+工业互联网"在PCB厂务场景的技术可行性与业务价值，为后续拓展至智能配电、零碳园区等全业务线奠定基础。

### 1.3 约束条件

- **时间约束**：30天交付可演示Demo
- **硬件约束**：单张入门级GPU（NVIDIA T4 16GB），64GB内存，1TB SSD
- **安全约束**：全量数据物理隔离，禁止任何形式的公网传输
- **人力约束**：3人核心小组（AI总工程师+算法工程师+后端工程师）

### 2.1 四层增强架构

采用**"基础设施-数据智能-业务引擎-交互管控"**四层架构，集成工业级评估体系与容错机制：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        【交互管控层】Interface Layer                       │
│   内部API网关 │ Web管理后台 │ 人工干预入口(Human-in-the-Loop) │ 审计日志   │
├─────────────────────────────────────────────────────────────────────────┤
│                        【业务引擎层】Agent Layer                           │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │              LangGraph 智能体状态机 (ReAct Pattern)                   │ │
│  │  意图识别 → 工具路由 → 执行监控 → 质量评估 → [置信度检查]              │ │
│  │                    ↓ 置信度<0.75或3次失败                             │ │
│  │              【人工Fallback节点】→ 审核队列 → 知识回流                  │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
├──────────────┬──────────────────┬──────────────────┬────────────────────┤
│ 【P1】知识问答 │ 【P2】报价复核     │ 【P3】合规审核      │ 【Eval】评估体系    │
│ RAG Engine   │ Quote Validator  │ Compliance Agent │ RAGAS+黄金测试集    │
├──────────────┴──────────────────┴──────────────────┴────────────────────┤
│                        【数据智能层】Data Intelligence                     │
│   Qdrant向量库 │ MinerU文档解析 │ 历史报价库 │ 知识库版本控制 │ 加密存储      │
├─────────────────────────────────────────────────────────────────────────┤
│                        【推理引擎层】Inference Engine                      │
```

## 第三章 核心功能详细设计

### 3.1 P1：企业专业知识库与AI智能问答系统

#### 3.1.1 知识库构建标准

**资料整合规范（首期300份）**
- **分类体系**：按"行业标准(GB/ISO)"+"设计手册(迪奥内部)"+"历史项目(PCB厂务案例)"+"供应商名录"四级分类
- **格式优先级**：Word/Excel(优先) → 标准PDF → 扫描件PDF(需OCR) → 图片
- **质量门槛**：MinerU解析后需通过**行列平衡校验**（针对表格类文档），确保数值型数据提取准确率>98%

**向量化策略**
```python
# 分块策略配置
ChunkConfig = {
    "chunk_size": 512,        # 保持暖通术语上下文
    "chunk_overlap": 100,     # 20%重叠防止语义断裂
    "separator": ["\n\n", "\n", "。", "；"],  # 中文语义边界
    "metadata_tags": ["document_type", "project_year", "system_type", "confidentiality_level"]
}

# 双重Embedding保障
Primary_Embedding: BAAI/bge-m3 (1024维，中文优化)
Backup_Embedding: text-embedding-3-small (1536维，兼容性备用)
```

#### 3.1.2 RAG增强生成链路

**完整流程**
```
用户Query → Query重写(术语扩展) → 向量检索(Top10) → 
重排序(Rerank: bge-reranker-v2-m3) → Top5精选 → 
上下文组装(System Prompt+检索结果+用户问题) → 
vLLM生成 → RAGAS实时评估 → 输出+置信度标签
```

**工业级评估体系（RAGAS）**
- **Faithfulness（忠实度）**：生成答案与检索上下文的逻辑一致性，阈值**≥0.85**
- **Answer Relevancy**：答案与问题匹配度，阈值**≥0.90**
- **Context Precision**：检索片段中相关句子占比，阈值**≥0.80**
- **Context Recall**：标准答案所需信息在检索结果中的覆盖率，阈值**≥0.75**

**黄金测试集（Golden Test Set）**
- **构建标准**：50道空调领域典型问题（L1基础15题+L2技术20题+L3推理15题），由AI总工程师与暖通专家共同标注标准答案与必备引用
- **回归机制**：每次Prompt修改或模型更新后强制测试，均值<0.85则阻断部署

### 3.2 P2：投标报价复核工具

#### 3.2.1 防幻觉设计原则

**硬逻辑隔离策略**
- **LLM职责**：字段映射建议、异常描述生成、调整建议草拟
- **Python硬编码职责**：所有数值计算（总价核算、均价统计、成本底线校验）
- **禁止行为**：LLM直接参与数学运算或价格预测

**三级校验机制**
```python
Validation_Rules = {
    "历史对标": "单品价格需在历史同型号均价±20%区间内",
    "成本底线": "分项报价≥(材料费+人工费+管理费)×1.03",
    "供应商白名单": "报价中供应商必须在Keycloak认证名录内"
}
```

**置信度标签系统**
- 🟢 **高置信（自动通过）**：三项校验全通过，历史匹配度>90%
- 🟡 **中置信（建议复核）**：两项通过，偏离度在20-30%区间
- 🔴 **低置信（强制人工）**：仅一项通过或偏离度>30%，自动转人工队列

#### 3.2.2 复杂表格解析方案

**MinerU专项优化**
- **适用场景**：多行合并单元格、跨页表格、表头嵌套
- **输出格式**：Markdown表格，保留合并逻辑（如`| 主机设备 | 冷水机组 | 100万 |`）
- **LLM理解增强**：针对Markdown表格设计专用System Prompt，明确"行标题/列标题/合并单元格"识别指令

**解析失败降级策略**
MinerU失败 → 自动切换Unstructured+规则修复 → 仍失败则标记"需人工录入" → 进入Fallback队列

### 3.3 P3：投标方案技术合规审核工具

#### 3.3.1 审核状态机（LangGraph）

```
[招标书上传] → [MinerU解析] → [条款结构化提取] → 
[合规项识别(ESG/能效/安全)] → [与投标方案比对] → 
[置信度评估] → {≥0.75?} → [生成审核报告]
                ↓ No
        [重试计数+1] → {≥3次?} → [人工Fallback节点]
                          ↓ No
                    [换用不同Prompt策略重试]
```

**人工Fallback触发条件**
1. 同一请求3次工具调用失败或返回空结果
2. 关键条款（如ESG合规、能效标准COP≥6.0）置信度<0.75
3. 单次审核耗时>30秒（疑似死循环）
4. 涉及金额>500万元的方案调整建议

**人工介入界面设计**
- 保留完整上下文：招标条款原文（高亮）+ AI推理链路 + 投标方案对应段落
- 操作选项："采纳AI建议" / "修正后采纳" / "完全人工重做"
- 知识回流：人工修正结果自动加入黄金测试集，用于后续模型优化

### 附录A：术语表

| 术语               | 解释                                                         |
| ------------------ | ------------------------------------------------------------ |
| **RAG**            | Retrieval-Augmented Generation，检索增强生成，指通过外挂知识库增强LLM回答准确性 |
| **AWQ/GPTQ**       | 大模型4bit量化技术，可将7B模型压缩至约5GB显存占用            |
| **Faithfulness**   | RAGAS评估指标，衡量生成内容是否忠实于检索到的上下文，防幻觉核心指标 |
| **Fallback**       | 当AI系统无法可靠完成任务时，自动转人工介入的容错机制         |
| **MinerU**         | 专门针对复杂版式PDF（含表格、公式）的开源解析工具，输出Markdown格式 |
| **PagedAttention** | vLLM核心优化技术，将KV Cache分页管理，显著降低显存碎片       |
