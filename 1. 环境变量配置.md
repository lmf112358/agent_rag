以下是项目需要手动配置的内容：

## 1. 环境变量配置

编辑 `backend/.env` 文件，需要配置：

| 配置项                | 说明                       | 示例值                |
| --------------------- | -------------------------- | --------------------- |
| **DASHSCOPE_API_KEY** | 通义千问API Key（必需）    | `sk-xxxxxx`           |
| **QDRANT_HOST**       | Qdrant向量库地址           | `localhost`           |
| **QDRANT_PORT**       | Qdrant端口                 | `6333`                |
| **MEMORY_TYPE**       | 记忆存储类型               | `inmemory` 或 `redis` |
| **REDIS_HOST**        | Redis主机（当使用Redis时） | `localhost`           |

## 2. 依赖安装

需要手动安装的依赖：

```bash
# 1. 安装后端依赖
cd backend
pip install -r requirements.txt

# 2. 安装核心RAG模块依赖（如果尚未安装）
cd ../langchain_rag
pip install -r requirements.txt
```

## 3. 外部服务配置

### Qdrant向量库
- 本地安装：`docker run -p 6333:6333 qdrant/qdrant`
- 或使用云服务：配置 `QDRANT_HOST` 为云服务地址

### Redis（可选，用于会话存储）
- 本地安装：`docker run -p 6379:6379 redis`
- 或使用云服务：配置 `REDIS_HOST`

## 4. 启动配置

### 启动顺序
1. 确保 Qdrant 服务已启动
2. 确保 Redis 服务已启动（如果使用）
3. 运行启动脚本：`python start.py`

### 端口配置
- 后端：8000
- 前端：3000
- Qdrant：6333
- Redis：6379

## 5. 数据准备

### 知识库构建
1. 准备文档放入 `data/` 目录
2. 使用 `langchain_rag/examples/quickstart.py` 中的 `example_05_document_processing()` 函数处理文档
3. 或直接调用 `QdrantVectorStore.from_documents()` 方法

### 历史价格数据
在 `backend/services/agent_service.py` 中配置历史价格数据：
```python
historical_prices={
    "冷水机组": 500000,
    "冷却塔": 80000,
    "风机盘管": 3500,
}
```

### 供应商白名单
```python
supplier_whitelist=["格力", "美的", "麦克维尔", "开利"]
```

## 6. 安全配置

- 生产环境需要修改 CORS 配置（`backend/main.py`）
- 生产环境需要配置 HTTPS
- 生产环境需要设置更复杂的会话ID生成策略

## 7. 性能配置

- 调整 `backend/config/settings.py` 中的 `max_session_length`
- 调整 `memory_session_ttl` 会话过期时间
- 调整 RAG 相关参数（`chunk_size`、`retrieval_top_k` 等）

## 8. 故障排查

- 检查 Qdrant 服务是否正常：`http://localhost:6333/dashboard`
- 检查 API 状态：`http://localhost:8000/health`
- 查看 API 文档：`http://localhost:8000/docs`
- 检查前端连接：`http://localhost:3000`

完成以上配置后，系统即可正常运行。
